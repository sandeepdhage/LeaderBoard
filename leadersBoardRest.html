post /match
records match result (addResult)
Adds match result item to the Leader Board's redis streams publishers and eventually redis database key value store. Client to this service ideally should be enhanced to post to Redis streams directly, going through rest API will have additional performance overhead while pushing updates for the match events.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body MatchResultItem (required)
Body Parameter — Match Result item to add

Responses

201

match result item added

400

invalid input, object invalid

409

an existing item already exists
Up 
get /leaders
search Leaders (searchLeaders)
Are per game mode (there are 4 of them) Have daily / weekly / all-time aggregations Contain the top 100 players

Query parameters

searchString (required)
Query Parameter — Pass an optional search string for looking up leaders - game mode , aggregations , locationId , group etc.
limit (optional)
Query Parameter — Maximum number of top players data to return format: int32

Return type

array[LeadersResults]
